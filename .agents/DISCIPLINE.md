# 紀律操作協議（Agent 輔助 × 你執行）

目標：你不靠當下感覺做決定；Agent 依規則產出「今日只做清單」，你只在約定時段執行。

## 分工

| 角色 | 負責 | 不負責 |
|------|------|--------|
| **Agent** | 抓報價／對停損／對 Level／產出清單與優先序 | 不下單、不保證獲利 |
| **你** | 在時段內依清單下單、回報成交結果 | 不盤中發明新理由、不攤平、不追價 |

## 每日節奏（平日）

1. **開盤前／早盤（可問 Agent）**：只要「今日清單」，不要討論感覺。
2. **09:00–09:30**：凍結，不市價砍倉。
3. **10:00–11:30**：執行「減碼／出清」限價單（反彈波）。
4. **13:40 後**：依收盤確認補做未完成項目；可問 Agent 更新。
5. **晚上（可選）**：美股／外匯／加密觀測，不下台股情緒單。

## 決策優先序（固定）

1. `force_exit` 出清窗（00632R、00882）— 大跌日優先賣反1  
2. 已成立「兩日不站回」的內資股 — 出清剩餘  
3. 外資主導股破防守 — 至少減半（2301）  
4. 其餘內資股 — 收盤確認／兩日法則，不盤中情緒砍  
5. 正2 — Level 3 **不加碼**；年線未破可續抱既有部位  
6. 出清資金 — Level 3 先抱現金，不急著買回

## 暫時焦點（大輸後修復期）

- **暫停**：IB／美股 ETF 新資金（匯款麻煩＋心理資本不足時不做）
- **可看**：台銀黃金存摺、比特幣（衛星倉，硬上限總資產 ≤5%）
- **原則**：只分批、只拉回買；不因台股賠錢情緒性一次滿倉避險資產
- 台股：維持既有剩餘持股規則；明日再看清單
- **通知**：`python src_scripts/scan_multi_asset.py --force`（黃金／BTC 買點與急跌）；台股停損用既有 `scan_black_swan` / `scan_position_levels` + Telegram/Email（填好 `api_keys.json` 並設 `NOTIFY_DRY_RUN: false`）

## 禁止清單

- 開盤殺盤市價出清（除非破產級流動性危機）
- Level 3 加碼正2／攤平個股
- 「基本面好」推翻已觸發的停損
- 一天內反覆改規則

## 你回報給 Agent 的格式（成交後貼上即可）

```
成交：
- 代號 / 張數 / 價格 / 剩餘張數
待確認：
- …
```

Agent 會據此更新 `config/my_targets.json` 建議與明日清單。
